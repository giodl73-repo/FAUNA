# FAUNA Agent Expansion Experiment

## Goal

Use agents to expand FAUNA from a seed animal-life schema into **50-100 candidate
capability systems** while improving the rubric after each batch.

## Batch contract

Each agent batch proposes 10-20 candidate systems. Every candidate must include:

| Field | Requirement |
|---|---|
| Capability id | Stable `fauna-capability:*` id. |
| Animal pressure | Herd, pest, predator, livestock, pollinator, scavenger, vector, migration, domestication, or habitat pressure. |
| Consumer fit | At least one BANISH, PORTO, CERES, LUCIA, or CANON use. |
| Rubric delta | Which FAUNA rubric axis the capability improves. |
| Non-goal | What repo should own the adjacent mechanics or interpretation. |

## First 10 capability lanes

1. Migration corridor pressure.
2. Predator fear and escort planning.
3. Livestock fodder and shelter debt.
4. Pest outbreak and storage loss.
5. Pollinator timing.
6. Disease-vector exposure.
7. Scavenger cleanup and contamination.
8. Draft-animal labor and injury.
9. Domestication memory and herd trust.
10. Habitat fragmentation from roads, farms, and sacred boundaries.

## Rubric improvement loop

After each batch, update [`QUALITY_RUBRIC.md`](..\QUALITY_RUBRIC.md) only when
the new candidates expose a repeated scoring gap. Do not add axes for one-off
ideas.
